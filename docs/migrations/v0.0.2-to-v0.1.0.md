# Migrate From Sanchika v0.0.2 To v0.1.0

## Installation

The following assets are available only after the v0.1.0 GitHub release is
published:

- `https://github.com/lamemustafa/sanchika/releases/download/v0.1.0/sanchika-tokens-0.1.0.tgz`
- `https://github.com/lamemustafa/sanchika/releases/download/v0.1.0/sanchika-primitives-0.1.0.tgz`
- `https://github.com/lamemustafa/sanchika/releases/download/v0.1.0/sanchika-patterns-0.1.0.tgz`

Declare all three direct tarballs and override the exact internal package
dependencies to the same GitHub assets. The overrides are required because
pnpm otherwise looks up the packed `0.1.0` dependency ranges on npm:

```json
{
  "dependencies": {
    "@sanchika/tokens": "https://github.com/lamemustafa/sanchika/releases/download/v0.1.0/sanchika-tokens-0.1.0.tgz",
    "@sanchika/primitives": "https://github.com/lamemustafa/sanchika/releases/download/v0.1.0/sanchika-primitives-0.1.0.tgz",
    "@sanchika/patterns": "https://github.com/lamemustafa/sanchika/releases/download/v0.1.0/sanchika-patterns-0.1.0.tgz"
  }
}
```

Add the exact internal dependency mappings to the consumer's
`pnpm-workspace.yaml`:

```yaml
overrides:
  "@sanchika/tokens@0.1.0": "https://github.com/lamemustafa/sanchika/releases/download/v0.1.0/sanchika-tokens-0.1.0.tgz"
  "@sanchika/primitives@0.1.0": "https://github.com/lamemustafa/sanchika/releases/download/v0.1.0/sanchika-primitives-0.1.0.tgz"
```

Then install and review the resulting lockfile:

```bash
pnpm install
```

These are GitHub release artifacts, not npm registry packages.

## CSS Order

```css
@import "@sanchika/tokens/theme.css";
@import "@sanchika/primitives/styles.css";
@import "@sanchika/patterns/styles.css"; /* only when product patterns are used */
@import "./consumer.css"; /* consumer CSS last */
```

## Compatibility Guidance

- Existing v0.0.2 CSS aliases continue to resolve during the documented
  compatibility window; new work should use the current semantic names.
- Existing `primitiveClassName` calls for `Button`, `Card`, `Badge`, and `Field`
  retain their valid legacy class output.
- The `PrimitiveSpec` shape and legacy TypeScript token collections remain
  supported.
- Existing legacy pattern exports remain supported. New product pattern APIs
  are additive and remain pre-1.0.
- `@sanchika/gallery-app` is a private deployed application, not a dependency
  or release package.
- Release tarballs report version 0.1.0. Source package manifests deliberately
  remain private at version 0.0.0 with `workspace:*` dependencies; the pack
  lane rewrites only clean temporary copies.

## Consumer Verification

Use public package entrypoints, not source paths:

```ts
import { tokenDefinitions } from "@sanchika/tokens";
import { formatIndianCurrency, primitiveClassName } from "@sanchika/primitives";
import { patternClassName } from "@sanchika/patterns";

const proof = {
  tokenCount: tokenDefinitions.length,
  amount: formatIndianCurrency(1248900),
  button: primitiveClassName("Button", "brand", "md"),
  reviewDesk: patternClassName("ReviewDeskPreview", {
    variant: "three-pane",
    state: "ca-review-needed",
  }),
};

if (!proof.button.includes("sk-button") || !proof.reviewDesk.includes("sk-pattern-review-desk-preview")) {
  throw new Error("Sanchika package-backed smoke failed");
}
```

Run the consumer's typecheck, build, browser smoke, and package-manager lockfile
check after installation. Record the three artifact checksums in adoption
evidence.

## Breaking And Changed Behavior

There is no intentional breaking change to the stable entrypoints
`@sanchika/tokens`, `@sanchika/primitives`, and `@sanchika/patterns`.

Material additive changes include a much larger semantic token vocabulary,
additional CSS for foundation/state/motion/product patterns, 28 primitive
contracts, 20 product pattern contracts, Indian formatting helpers, and
generated gallery documentation. Because this is pre-1.0, review CSS cascade,
new contract use, output size, and consumer-owned interaction behavior rather
than assuming all future minor releases will be additive.

## Rollback To v0.0.2

Restore these exact assets together:

- `https://github.com/lamemustafa/sanchika/releases/download/v0.0.2/sanchika-tokens-0.0.2.tgz`
- `https://github.com/lamemustafa/sanchika/releases/download/v0.0.2/sanchika-primitives-0.0.2.tgz`
- `https://github.com/lamemustafa/sanchika/releases/download/v0.0.2/sanchika-patterns-0.0.2.tgz`
- `https://github.com/lamemustafa/sanchika/releases/download/v0.0.2/manifest.json`

1. Replace the three direct dependency URLs and both `pnpm-workspace.yaml`
   override values with
   the v0.0.2 URLs above.
2. Change the pnpm override selectors to `@sanchika/tokens@0.0.2` and
   `@sanchika/primitives@0.0.2`; leaving the v0.1.0 selectors would allow the
   rollback dependencies to fall through to npm.
3. Remove code and CSS usage that depends on v0.1.0-only APIs.
4. Run `pnpm install` to regenerate the lockfile, then review the lockfile diff
   and confirm all three Sanchika packages resolve to 0.0.2.
5. Run the consumer package-backed smoke, typecheck, build, and browser checks.
6. Commit the consumer manifest and lockfile rollback together.

Rollback affects package artifacts only. It does not require a database, DNS,
or workspace migration.
