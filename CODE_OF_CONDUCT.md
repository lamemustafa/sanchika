# Code of Conduct

Sanchika is a public design-system SDK for compliance-grade interfaces. Project
discussion must stay professional, evidence-based, and safe for contributors
working around sensitive compliance domains.

## Expected Behavior

- Be respectful and direct.
- Use synthetic examples instead of real taxpayer, credential, portal, or local
  machine data.
- Keep disagreement focused on code, docs, accessibility evidence, and release
  risk.
- Report security issues and sensitive data exposure through private channels,
  not public issues or pull requests.

## Unacceptable Behavior

- Harassment, threats, insults, sustained disruption, or other harassment.
- Publishing private or sensitive data, including PAN, GSTIN, Aadhaar,
  credentials, cookies, OTPs, notices, screenshots, local paths, or taxpayer
  names.
- Misrepresenting Sanchika as production-ready, official, government-approved,
  filing-ready, or security-audited without repository evidence.

## Enforcement

Maintainers may edit, hide, or remove comments and close issues or pull requests
that violate this Code of Conduct. Repeated or severe violations may lead to a
block from the repository.

To report conduct concerns, contact the repository owner privately. If the
concern is also a vulnerability or sensitive-data exposure, use GitHub private
vulnerability reporting once it is enabled for `lamemustafa/sanchika`.
