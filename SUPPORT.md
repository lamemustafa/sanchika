# Support

Sanchika support is limited to the public SDK contracts in this repository:
tokens, primitives, patterns, gallery proof artifacts, and adoption guidance.

## Public Questions

Use the question issue template for non-sensitive SDK questions that can be
discussed publicly. Keep examples synthetic.

Good public questions include:

- How to consume a token, primitive, or pattern contract.
- How to interpret an accessibility or trust-boundary requirement.
- How to plan a safe consumer adoption proof.

Do not use Sanchika issues for ComplyEaze app support, Pack portal debugging,
tax or legal advice, filing decisions, taxpayer-specific review, production
incidents, credentials, or downloaded portal artifacts.

## Bugs And Features

- Use the bug report template for reproducible SDK defects.
- Use the feature request template for proposed SDK behavior or documentation
  changes.
- Use the conduct report template only for non-sensitive Code of Conduct
  concerns that can be discussed publicly.
- Send sensitive conduct concerns that are not vulnerabilities to
  `security@complyeaze.com` with the subject `Sanchika conduct report`.

## Sensitive Material

Do not post PAN, GSTIN, Aadhaar, credentials, cookies, OTPs, notices,
screenshots, local file paths, private messages, taxpayer names, or real taxpayer
data in public issues, pull requests, discussions, commits, screenshots, or
examples.

Use GitHub private vulnerability reporting for suspected vulnerabilities or
sensitive-data exposure.
